1. Project Overview

This submission delivers Part 3 of the GLMS platform. The monolith from Part 2 has been refactored into a Service-Oriented Architecture (SOA) with three Docker containers: SQL Server database, ASP.NET Core Web API backend, and ASP.NET Core MVC frontend.

2. Solution Structure

ProjectRole
GLMS.SharedShared models and DTOs
GLMS.ApiWeb API — database, business logic, JWT auth, Swagger
EAPD7111_PART2MVC frontend — calls API via HttpClient only
EAPD7111_PART2.TestsUnit tests + API integration tests

3. How to Run with Docker

  1. Install Docker Desktop
  2. Open terminal in project folder
  3. Run: docker compose up --build
  4. Open browser: http://localhost:8081 (MVC frontend)
  5. API Swagger: http://localhost:8080/swagger
  6. Login: username admin, password Admin@123

4. API Endpoints (Summary)

  • POST /api/auth/login — JWT authentication
  • GET/POST/PUT/DELETE /api/clients — Client management
  • GET/POST/PATCH/DELETE /api/contracts — Contract management with filtering
  • PATCH /api/contracts/{id}/status — Approve/decline contract status
  • GET/POST/PUT/DELETE /api/servicerequests — Service requests with USD→ZAR
  • GET /api/servicerequests/exchange-rate — Live exchange rate

5. Testing

Run unit and integration tests in Visual Studio Test Explorer or:

dotnet test EAPD7111_PART2.Tests/EAPD7111_PART2.Tests.csproj

Integration tests call live API endpoints using WebApplicationFactory and assert HTTP 200/201 status codes and data integrity.
